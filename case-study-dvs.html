A scalable visualization framework powering insights across Rapid7's multi-product platform

Executive Summary

Rapid7's platform provides analysts, security leaders, and executives with critical telemetry, yet each product team had independently built dashboards, charts, and reporting tools. This fragmentation created inconsistent experiences, duplicated engineering investment, and made insight discovery slower and more cognitively taxing for users.

I led the strategy, architecture, and design of the Data Visualization System (DVS)—a unified visualization framework enabling reusable dashboards, card components, reporting templates, and cross-product filtering. Together with engineering and design partners, we delivered a scalable system powering analytics across six enterprise security products, significantly improving usability, performance, and design/engineering efficiency.

The Challenge

Rapid7's products—InsightIDR, InsightVM, AppSec, Connect, Endpoint, and Threat Command—each had their own approaches to data visualization, creating significant friction:

  • Different chart types and visualization paradigms
  • Different reporting systems and export capabilities
  • Different card styles and interaction patterns
  • Different filtering logic across products
  • No shared visualization language or standards
  • Duplicated engineering effort (6+ implementations of dashboards & reports)
  • Inconsistent charts leading to misinterpretation of trends
  • Limited accessibility in charts and data views

This inconsistency made it difficult for users to move between product areas, compare metrics, or build a mental model of the platform.

Research & Insights

I synthesized insights across field research, interviews, analytics usage, and cross-team workshops to understand the needs of different user personas.

Key User Insights by Persona

Analysts:

  • "I need to know what's happening right now—I don't have time to decode visuals."
  • Value fast triage, anomalies, and drill-downs

Managers:

  • Want clarity on severity, coverage, and progress
  • Need to communicate status to leadership quickly

Executives:

  • Care about trends, ROI, and posture change
  • Need high-level summaries that tell a story

Cross-Product Synthesis

  • Users expect dashboards to behave the same across products
  • Inconsistent filtering increases cognitive load
  • Cards need to communicate insight before interaction
  • Drill-in flows must support multi-step investigations
  • Empty/loading states were a major source of confusion

DVS North Stars

From this research, I derived three guiding principles:

  • Temperature Read — Show what matters most at a glance
  • Investigate — Support exploration with clarity and consistency
  • Take Action — Drive users toward next steps

Strategy & Architecture

I architected DVS around three interoperable layers:

Layer 1 — Templates (Dashboards & Reports)

  • Overview dashboards for at-a-glance monitoring
  • Custom dashboards for personalized views
  • Cross-product Home experiences
  • Report templates (PDF + print-optimized theme)

Layer 2 — Card System (Composable Visualization Components)

Each card type was designed to be self-contained and reusable:

  • KPI cards for key metrics
  • Trend cards for time-series data
  • Table cards for detailed listings
  • Top-N lists for rankings
  • Correlation/relationship cards
  • Timeline/sequence cards
  • Investigation cards for drill-downs

Each card defined:

  • Inputs and data requirements
  • Visualization type and configuration
  • Behavior and interaction patterns
  • States (empty, loading, error)
  • Drill-in model
  • Accessibility rules and requirements

Layer 3 — Filtering, Querying & Drill-In Logic

  • Global filter model affecting all cards
  • Local filters for card-specific refinement
  • Query chips for transparent filter management
  • Time range patterns
  • Cross-product drill-in framework

Design Principles

Five core principles guided all design decisions:

  • Consistency Over Novelty — Every chart follows a consistent visual language
  • Clarity of Insight — Cards must communicate value at a glance
  • Accessible Visuals — High-contrast color palette, screen-reader friendly load states
  • Composable Components — Everything must be reusable across contexts
  • Performance-Oriented — Designed for enterprise-scale data volume

Design & Implementation

Component Design & Figma Library

  • Built a full Figma card library with reusable structure
  • Defined slot-based card architecture for flexibility
  • Created atoms → molecules → card-level components
  • Integrated DVS tokens into RDS design system

Engineering Collaboration

Together with engineering, we:

  • Created shared data adapters across products
  • Standardized charting library configuration
  • Improved performance (avoiding redundant re-renders)
  • Implemented consistent drill-in flows

Reporting System

I designed a comprehensive reporting framework including:

  • Print-optimized theme
  • Layout templates for different report types
  • Cover pages and branding
  • Typographic hierarchy for readability
  • Page-break logic for multi-page reports
  • Export UX for PDF generation
